(async, static) signIn(req, res) → {Object}

Description:
  • POST /api/signin
    This function is used to sign in a user. It first checks if the user exists in the database using checkExistingUser function. If the user exists, it compares the password provided by the user with the hash of password stored in the database using the compare function from the utils folder. If the password is correct, it sends a success response with the username in the response body and a JWT token in the response headers. If username and passord doesn't match the Joi schema, it sends a failure response with the appropriate message in the response body. This is done in the verifyRequest from the middlewares folder. For response handling, it uses the responseBuilder function from the utils folder. It is wrapped in a try-catch block to catch any errors. In case of an error, it logs the error and sends a generic failure response.

(async, static) signUp(req, res) → {Object}

Description:
  • POST /api/signup
    This function is used to sign up a user. It first checks if the user exists in the database using checkExistingUser function. If the user exists, it sends a failure response. If username, password and email (optional) doesn't match the Joi schema, it sends a failure response with the appropriate message in the response body. This is done in the verifyRequest from the middlewares folder. For response handling, it uses the responseBuilder function from the utils folder. It is wrapped in a try-catch block to catch any errors. In case of an error, it logs the error and sends a generic failure response.
