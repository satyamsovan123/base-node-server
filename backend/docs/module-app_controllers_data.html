(async, static) getData(req, res) → {Object}

Description:
  • GET /api/data?offset=0
    This function is used to get all the data from the database. It uses the pagination options from the paginationOptions file and the sanitizeURLQueryParam function to sanitize the offset from the query params. If no data is found, it sends a failure response. For response handling, it uses the responseBuilder function from the utils folder. It is wrapped in a try-catch block to catch any errors. In case of an error, it logs the error and sends a generic failure response.

(async, static) getRandomData(req, res) → {Object}

Description:
  • GET /api/random-data
    This function is used to get a random data from the database. This endpoint does not use pagination and doen not uses verifyJWT middleware. If no data is found, it sends a failure response. For response handling, it uses the responseBuilder function from the utils folder. It is wrapped in a try-catch block to catch any errors. (async, static) postData(req, res) → {Object}

Description:
  • POST /api/data
    This function is used to post data to the database. If title and article doesn't match the Joi schema, it sends a failure response with the appropriate message in the response body. This is done in the verifyRequest from the middlewares folder. For response handling, it uses the responseBuilder function from the utils folder. It is wrapped in a try-catch block to catch any errors. In case of an error, it logs the error and sends a generic failure response.
