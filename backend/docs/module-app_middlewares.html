(async, static) verifyJWT(req, res, next) → {Object}

Description:
  • This middleware function is used for verifying the JWT token for a route. It uses the JWT_SECRET_KEY from the .env file to verify the token. It also checks if the user exists in the database. If the user exists, it adds the username to the request body. If the user does not exist, it sends a generic failure response. For response handling, it uses the responseBuilder function from the utils folder. It is wrapped in a try-catch block to catch any errors. In case of an error, it logs the error and sends a generic failure response.

(async, static) verifyRequest(req, res, next) → {Object}

Description:
  • This function validates the request body for different API endpoints. It uses the Joi library for validation. It uses different Joi schemas like signInValidatorSchema, signUpValidatorSchema, dataValidatorSchema from the models folder for different API endpoints. For response, it uses the responseBuilder function from the utils folder. It is wrapped in a try-catch block to catch any errors. In case of an error, it sends back the error message in the response.
